# Agent简介

Agent，又称为智能体。借助LLM强大的推理和规划能力，通过使用外部工具调用来执行规划好的任务，Agent能够自主地、智能地完成各种各样的复杂任务，例如：写代码、自动修复bug等场景。

> Agent以LLM作为大脑进行任务规划，通过Action来执行具体的任务，使用Memory来获取、存储以及检索信息。

我对Agent的认识如下：Agent是一个系统性的工程项目。既需要丰富的工程经验，能够结合LLM的特性设计出可扩展的、容错性的、分布式的系统架构；也需要了解LLM、提示工程等技术，了解数据工程、业务领域知识(API设计)。

作为一个LLM-based Agent，其设计目标应始终对人类有益，也就是说，人类可以利用人工智能造福人类。具体来说，我们希望Agent能实现以下目标：
帮助用户从日常任务和重复劳动中解脱出来，从而减轻人类的工作压力，提高任务解决效率。
不再需要用户提供明确的低级指令。相反，Agent可以独立分析、规划和解决问题。
在解放用户双手的同时，Agent也解放了他们的大脑，使其能够从事探索性和创新性工作。
